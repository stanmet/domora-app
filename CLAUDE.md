# CLAUDE.md · Проект Domora

Ты помогаешь непрограммисту построить маркетплейс услуг Domora (Ирландия).
Владелец не пишет код: объясняй действия простыми словами, все команды запускай сам,
после каждой задачи проверяй результат запуском и говори, что смотреть в браузере.

## Что это за продукт
Маркетплейс бытовых услуг: повар на дом, уборка, мастер, массаж, красота, праздники.
Исполнители сами создают плашку услуги (профессия + название + цена + единица), модерация проверяет.
Деньги: холд на карте при запросе, списание при подтверждении исполнителем,
выплата исполнителю через 24 часа после завершения, если нет спора.
Полная логика: файл docs/domora-spec.md (споры, отмены, страйки, автоперевод чата на 6 языков).

## Ключевые файлы репозитория
- docs/domora-spec.md - продуктовая спецификация, источник правды по логике
- prisma/schema.prisma - готовая схема БД, менять только по согласованию
- src/lib/stripe.ts, src/lib/jobs.ts, src/app/api/** - готовое платежное ядро
- prototypes/Marketplace.jsx, prototypes/HostDashboard.jsx - утвержденный UI и дизайн-система,
  переноси разметку и стили из них, дизайн не переизобретать

## Технические правила
- Next.js App Router + TypeScript, Prisma + PostgreSQL (Supabase), Supabase Auth
- Все суммы в центах (Int), любые изменения статуса брони пишутся в booking_events
- Stripe: manual capture, separate charges and transfers, вебхуки - источник правды
- Языки интерфейса: en ru uk pl es pt; тексты только через словари, без хардкода
- В текстах интерфейса не использовать длинное тире
- Секреты только в .env, в код не вписывать
- После каждой фичи: npm run build должен проходить без ошибок

## Спринт 1 (текущий): фундамент
1. Инициализировать Next.js проект, подключить Prisma к Supabase, прогнать миграцию schema.prisma
2. Seed-скрипт: категории (chef, clean, handy, massage, beauty, events, other) и 3 тестовых исполнителя
3. Supabase Auth: вход по email (magic link) и Google, страницы входа для клиента и исполнителя
4. Перенести из prototypes/Marketplace.jsx: главную, каталог с поиском и вкладками, профиль исполнителя;
   данные брать из БД через Prisma, а не из моков
5. Деплой на Vercel, проверка с телефона
